# PM 状态→角色映射表

本文件由 `~/.claude/agents/pm.md` 的 SOP 段落外部化而来。PM 和其他需要调度判断的 agent 均引用本文件。

## 状态到角色的默认映射

| Task 状态 | 目标 Agent | 备注 |
|-----------|-----------|------|
| "需求已明确" | `开发组长`（普通开发）/ `机器学习工程师`（纯 ML）/ `深度研究员`（研究）/ `技术调研师`（调研）/ `创意策划师`（创意） | 按任务类型分流 |
| "方案设计中" + 架构升级 | `架构师` | |
| "方案设计中" + 数据库 | `数据库工程师` | |
| "方案设计中" + 视觉系统 | `视觉设计师` | |
| "方案已完成" | `后端开发师` / `前端开发师` / `机器学习工程师` / `文档工程师` / `运维部署工程师` | 按实现类型分流 |
| "开发完成待审查" | `代码审计师` | |
| "审查未通过-待修复" | 对应开发 agent | 回到原实现者 |
| "审查通过待测试" | `功能测试师` → `界面测试师`（涉前端）→ `安全审计师`（上线前）→ `测试总监师` | 串行质量链 |
| "测试未通过-待修复" | 对应开发 agent（回到审查） | 修复后重走审查 |
| "上线前必检" | `安全审计师`（深度）+ `测试总监师`（裁决） | |

## test-ui 跳过条件

**同时满足以下全部条件时可跳过 test-ui**，并在调度记录的"特殊说明"写明跳过理由：

1. Task 改动不涉及前端界面
2. 纯后端 API / 纯配置 / 纯文档 / 纯 ML 推理
3. 已有功能测试覆盖证明业务正确性

## security-auditor 触发条件

以下任一条件满足时触发 security-auditor：

1. **里程碑前**：大版本、M1/M2 交付前
2. **敏感功能上线前**：涉及金钱、健康、PII、凭据的功能上线前
3. **升级触发**：code-review 发现疑似深层安全问题时（不自己做，升级到 security-auditor）
