```
| 评估维度 | 权重 | 候选 A | 候选 B | 候选 C | 评分说明 |
|---------|------|--------|--------|--------|----------|
| 功能覆盖度 | 25% | | | | 针对具体用例的功能匹配度 |
| 总拥有成本 | 25% | | | | 12 个月 TCO，含集成和运维 |
| 集成复杂度 | 20% | | | | 工程天数 + 学习曲线 |
| 风险 profile | 15% | | | | 许可证 + 供应商稳定性 |
| 社区/支持 | 15% | | | | 文档质量 + 响应速度 |
| **加权总分** | 100% | | | | |
```

### 评分标准（1-5 分制）

| 分数 | 功能覆盖度 | 成本 | 集成复杂度 | 风险 | 社区 |
|------|-----------|------|-----------|------|------|
| 5 | 完全覆盖，有增值功能 | 显著低于预算 | < 2 天，SDK 优秀 | 无风险，MIT/Apache | 活跃，官方响应快 |
| 4 | 核心功能覆盖 | 预算内 | 2-5 天，SDK 良好 | 低风险，LGPL | 较活跃 |
| 3 | 大部分覆盖，需 workaround | 略超预算 | 5-10 天，SDK 一般 | 中等风险，需关注 | 一般 |
| 2 | 部分覆盖，需大量定制 | 显著超预算 | 10-20 天，SDK 差 | 高风险，AGPL/SSPL | 不活跃 |
| 1 | 基本不满足 | 不可接受 | > 20 天或不可用 | 极高风险 | 无人维护 |

## 1.2 定价对比表模板

### SaaS 服务定价分析

```
## Pricing Analysis: [Service Name]
**Analysis Date**: [YYYY-MM-DD]
**Source**: [URL]

### 定价结构
| 层级 | 月费 | 包含量 | 超出单价 | 关键限制 |
|------|------|--------|----------|----------|
| Free | $0 | 100 req/day | N/A | 无 SLA，社区支持 |
| Starter | $29 | 10k req/month | $0.01/req | 99.9% SLA |
| Pro | $99 | 100k req/month | $0.005/req | 99.95% SLA |
| Enterprise | 定制 | 定制 | 定制 | 99.99% SLA |

### 成本估算（12 个月）
| 场景 | 月用量 | 月费 | 年费 | 备注 |
|------|--------|------|------|------|
| 当前 | 500/day | $0 (Free) | $0 | |
| 6 个月预测 | 5k/day | $29 | $348 | 接近 Starter 上限 |
| 12 个月预测 | 20k/day | $99 | $1,188 | 需 Pro 层级 |
| 峰值场景 | 50k/day | $99 + $500 | $7,188 | 超出 Pro 包含量 |

### 隐藏成本
- **超量费用**: 超出包含量后 $0.005/req，峰值月可能产生 $500+ 额外费用
- **年度折扣**: 年付 8 折，但需预付
- **支持费用**: 电话支持需 Enterprise 层级
- **数据导出**: 批量导出收费 $0.01/record

### 定价历史
- 2024-03: 涨价 20%，Free 层级从 500/day 降至 100/day
- 2025-01: 新增 Pro 层级
- 风险: 过去 2 年涨价 2 次，趋势向上
```

### API 服务定价分析（按调用量）

```
## API Pricing: [Service Name]
**Analysis Date**: [YYYY-MM-DD]

| 服务 | 输入单价 | 输出单价 | 上下文窗口 | RPM 限制 | TPM 限制 |
|------|----------|----------|------------|----------|----------|
| GPT-4o | $5/1M tokens | $15/1M tokens | 128K | 500 | 30K |
| Claude 3.5 Sonnet | $3/1M tokens | $15/1M tokens | 200K | 400 | 40K |
| Gemini 1.5 Pro | $3.5/1M tokens | $10.5/1M tokens | 1M | 360 | 60K |

### 月度成本估算（假设 100K 输入 + 50K 输出 tokens/天）
| 服务 | 日输入成本 | 日输出成本 | 月成本 | 年成本 |
|------|-----------|-----------|--------|--------|
| GPT-4o | $0.50 | $0.75 | $37.50 | $450 |
| Claude 3.5 | $0.30 | $0.75 | $31.50 | $378 |
| Gemini 1.5 | $0.35 | $0.525 | $26.25 | $315 |

### 配额/限制分析
- **RPM (Requests Per Minute)**: 100K tokens/req ≈ 5 req/min，远低于 360-500 RPM 限制
- **TPM (Tokens Per Minute)**: 150K tokens/min，接近 Claude 的 40K 限制，需批处理
- **并发**: 所有服务支持 100+ 并发，无瓶颈
```

## 1.3 定价陷阱识别清单

### Freemium 陷阱
- [ ] Free  tier 是否包含核心功能？
- [ ] Free  tier 的限制是否会在项目早期就触发？
- [ ] 从 Free 到 Paid 的过渡是否平滑？
- [ ] 是否存在"免费试用后自动扣费"的陷阱？

### 计量计费陷阱
- [ ] 计费单位是否清晰？（per request / per token / per GB / per seat）
- [ ] 是否存在"最低消费"？
- [ ] 峰值用量是否会导致账单激增？
- [ ] 是否有用量预警机制？

### 企业级陷阱
- [ ] Enterprise 层级是否有"联系销售"才能获取的隐藏功能？
- [ ] 年度合同是否有自动续约条款？
- [ ] 提前终止是否有违约金？
- [ ] 数据导出是否在合同终止后收费？
