# 前端开发师 — Accessibility (A11y) Baseline

## A11y Is an Implementation Requirement, Not an Enhancement

Keyboard navigation, focus rings, ARIA attributes, and contrast ratios are implementation requirements, present in the initial delivery. Not an enhancement, not a nice-to-have, not a post-launch item.

The A11y Afterthought anti-pattern: implementing full visual design and interaction, then attempting to "add accessibility" as a final step. A custom dropdown built with `<div>` elements that then has `role="button"` added but still doesn't handle Arrow key navigation or focus restoration.

**Correction**: start with semantic HTML; reference ARIA Authoring Practices Guide for custom interactive patterns.

---

## A11y Self-Check — 6 Items (Run Before Recommending @code-review)

1. **Keyboard navigation**: Tab through every interactive element — can all controls be reached and activated by keyboard only?
2. **Focus rings**: Are focus rings visible on all interactive elements? (`focus-visible:ring-2` or equivalent)
3. **Alt text**: Do all `<img>` elements have meaningful `alt` attributes? (decorative images: `alt=""`)
4. **Form labels**: Does every `<input>`, `<select>`, `<textarea>` have a corresponding `<label>` or `aria-label`?
5. **aria-live regions**: Do dynamic content updates (async data loads, form errors, notifications) use `aria-live`?
6. **Contrast ratios**: Body text ≥ 4.5:1, large text and UI components ≥ 3:1 (WCAG 2.1 AA)

## Domain 2.3.1 — Keyboard Navigation Contract

```
Every interactive element reachable by Tab in logical order
Tab trap for modal dialogs (focus must not escape while open)
Escape closes modals and dropdowns
Arrow keys navigate within list/grid semantics
Keyboard path must be as efficient as mouse path
```

**Tab order management**

```tsx
// BAD: custom button with <div> — not keyboard accessible
<div onClick={handleClick} className="btn-primary">Send</div>

// GOOD: semantic button — keyboard accessible by default
<button type="button" onClick={handleClick} className="btn-primary">Send</button>

// BAD: tabIndex=-1 on interactive elements hides them from keyboard users
<button tabIndex={-1} onClick={handleClick}>Submit</button>

// GOOD: tabIndex=-1 only for elements that should be programmatically focused but not in tab order
// (e.g., modal container that receives focus via JS, not sequential tab)
```

**Modal Tab Trap**

```tsx
import { useEffect, useRef } from 'react';

function Modal({ isOpen, onClose, children }: ModalProps) {
    const modalRef = useRef<HTMLDivElement>(null);

    useEffect(() => {
        if (!isOpen) return;

        // Focus first focusable element when modal opens
        const firstFocusable = modalRef.current?.querySelector<HTMLElement>(
            'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
        );
        firstFocusable?.focus();

        // Trap focus within modal
        const handleKeyDown = (e: KeyboardEvent) => {
            if (e.key === 'Escape') {
                onClose();
                return;
            }
            if (e.key !== 'Tab') return;

            const focusableElements = modalRef.current?.querySelectorAll<HTMLElement>(
                'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
            );
            if (!focusableElements || focusableElements.length === 0) return;

            const first = focusableElements[0];
            const last = focusableElements[focusableElements.length - 1];

            if (e.shiftKey && document.activeElement === first) {
                e.preventDefault();
                last.focus();
            } else if (!e.shiftKey && document.activeElement === last) {
                e.preventDefault();
                first.focus();
            }
        };

        document.addEventListener('keydown', handleKeyDown);
        return () => document.removeEventListener('keydown', handleKeyDown);
    }, [isOpen, onClose]);

    if (!isOpen) return null;

    return (
        <div
            ref={modalRef}
            role="dialog"
            aria-modal="true"
            aria-labelledby="modal-title"
        >
            {children}
        </div>
    );
}
```

## Domain 2.3.2 — ARIA Implementation Precision

```
role — only when semantic HTML doesn't cover the case
aria-label — when accessible name cannot be derived from content
aria-describedby — for descriptions (not names)
aria-live="polite" — for non-urgent updates
aria-live="assertive" — ONLY for urgent updates (errors that block user)
aria-expanded, aria-selected, aria-checked — for interactive states
```

**ARIA rules**

```tsx
// BAD: role on element that already has semantics
<button role="button">Submit</button>

// BAD: aria-label when visible text already describes it
<button aria-label="Submit form">Submit</button>

// GOOD: aria-label when icon button has no visible text
<button aria-label="Close dialog" onClick={onClose}>
    <CloseIcon aria-hidden="true" />
</button>

// GOOD: aria-live for dynamic content (invitation list updates)
<div aria-live="polite" aria-atomic="false">
    {/* Content updates here will be announced to screen reader */}
    {invitations.length} invitations
</div>

// GOOD: aria-expanded for custom disclosure widget
<button
    aria-expanded={isOpen}
    aria-controls="dropdown-menu"
    onClick={() => setIsOpen(!isOpen)}
>
    Actions
</button>
<ul id="dropdown-menu" hidden={!isOpen} role="menu">
    {/* menu items */}
</ul>
```

## Domain 2.3.3 — Focus Management for SPA Navigation

```
On route change: move focus to main content or <h1>
When modal opens: move focus to first focusable element inside
When modal closes: return focus to trigger element
When form field error appears: do NOT move focus automatically
```

**Route change focus management**

```tsx
// React Router — focus main content on route change
import { useLocation } from 'react-router-dom';
import { useEffect, useRef } from 'react';

function App() {
    const location = useLocation();
    const mainRef = useRef<HTMLElement>(null);

    useEffect(() => {
        // Move focus to main content on route change
        mainRef.current?.focus();
    }, [location.pathname]);

    return (
        <main ref={mainRef} tabIndex={-1} id="main-content">
            {/* Route content */}
        </main>
    );
}
```

**Return focus to trigger when modal closes**

```tsx
function useModalFocusReturn() {
    const triggerRef = useRef<HTMLElement | null>(null);

    const openModal = () => {
        // Save current focus before opening
        triggerRef.current = document.activeElement as HTMLElement;
        setIsOpen(true);
    };

    const closeModal = () => {
        setIsOpen(false);
        // Restore focus to trigger element
        triggerRef.current?.focus();
    };

    return { openModal, closeModal };
}
```

## Contrast Ratio Reference

| Text type | Minimum ratio | WCAG level |
|---|---|---|
| Body text (< 18pt) | 4.5:1 | AA |
| Large text (≥ 18pt / ≥ 14pt bold) | 3:1 | AA |
| UI components (borders, icons) | 3:1 | AA |
| Disabled controls | No minimum | (exempt) |
| Decorative elements | No minimum | (exempt) |

**Verification tool**: browser DevTools accessibility panel, or axe DevTools extension.
