# Domain 1: 命名方法论与 Slogan 生成框架

## 1.1 SMILE 命名框架（扩展版）

SMILE 是一个结构化的命名评估框架，确保每个候选名称在五个维度上都经过检验：

### S — Strategic Fit（战略契合）

**检验问题**：
- 这个名称是否支持品牌的长期战略定位？
- 是否能够在品牌扩展时仍然适用？
- 是否与品牌的核心价值链相符？

**评分标准**：
| 分数 | 标准 |
|------|------|
| 5 | 完美契合品牌战略，可支撑 5 年以上发展 |
| 4 | 高度契合，可支撑 3-5 年发展 |
| 3 | 基本契合，但扩展性有限 |
| 2 | 部分契合，可能需要中期更名 |
| 1 | 与品牌战略不符 |

### M — Memorability（记忆度）

**检验方法**：
1. **隔夜测试**: 向目标用户展示名称，12 小时后无提示回忆
2. **电话测试**: 在嘈杂环境中说出名称，对方能否正确拼写？
3. **搜索测试**: 在 Google/Baidu 搜索该名称，是否容易找到品牌？

**评分标准**：
| 分数 | 标准 |
|------|------|
| 5 | 90%+ 目标用户能在一周后回忆 |
| 4 | 70-90% 能回忆 |
| 3 | 50-70% 能回忆 |
| 2 | 30-50% 能回忆 |
| 1 | <30% 能回忆 |

### I — Intonation & Sound（音韵）

**检验维度**：
- **音节数**: 中文 ≤4 音节，英文 ≤3 音节为佳
- **发音难度**: 是否包含难发音的音素组合？
- **语调**: 中文四声组合是否和谐？
- **跨语言**: 在其他目标市场语言中是否易发音？

**常见音韵陷阱**：
- 连续三个第三声（如 "李雨宇"）→ 拗口
- 英文中以 "-th" 结尾 → 部分亚洲用户难发音
- 包含 "r" 和 "l" 的复杂组合 → 易混淆

### L — Legal Safety（法律安全）

**检查清单**：
- [ ] 商标初步搜索（CNIPA/USPTO/EUIPO）
- [ ] 域名可用性（.com / .cn / .io / .ai）
- [ ] App Store / 应用商店搜索
- [ ] 社交媒体账号可用性
- [ ] 同行业已知品牌碰撞

**风险等级**：
| 等级 | 说明 | 行动 |
|------|------|------|
| 绿色 | 初步搜索无冲突 | 可继续，但仍建议正式查询 |
| 黄色 | 有近似商标，但不同类别 | 需法律评估 |
| 红色 | 有直接冲突 | 放弃该候选 |

### E — Emotional Resonance（情感共鸣）

**检验问题**：
- 目标用户听到这个名称时的第一反应是什么？
- 是否唤起了品牌希望关联的情感？
- 是否有意外的负面情感联想？

**测试方法**：
1. 向 5-10 个目标用户展示名称（不提供上下文）
2. 记录他们的第一反应词（3 个形容词）
3. 分析是否与品牌定位一致

## 1.2 七框架命名法详解

### 框架 1: Descriptive（描述型）

**特征**: 直接描述产品功能或类别
**优点**: 立即传达价值主张，SEO 友好
**缺点**: 难以注册商标，可能显得平庸
**适用**: 功能性强的 B2B 产品

**示例**: 
- Runbook（运维手册 → 工程管理工具）
- DataSync（数据同步）
- CloudStorage（云存储）

**生成方法**: 
1. 列出产品核心功能的动词和名词
2. 组合: [功能动词] + [功能名词] 或 [核心名词] + [类别词]
3. 检查是否过于直白（如 "BestProjectManagementTool"）

### 框架 2: Evocative（ evocative 型）

**特征**: 通过隐喻和联想唤起情感
**优点**: 富有诗意，品牌延展性强
**缺点**: 需要营销投入来建立关联
**适用**: 情感驱动的 B2C 品牌

**示例**:
- Anchor（锚 → 稳定性）
- Strand（线 → 专注、连接）
- Horizon（地平线 → 视野、未来）

**生成方法**:
1. 确定品牌希望唤起的核心情感
2. 列出与该情感相关的自然/物理/抽象概念
3. 选择具有正面联想且不易过时的概念

### 框架 3: Coined（创造型）

**特征**: 发明的新词，控制音韵
**优点**: 独特，易于注册商标
**缺点**: 需要建立认知，初期 SEO 困难
**适用**: 创新产品，追求独特性的品牌

**示例**:
- Lumio（光 + io）
- Drydock（干船坞 → 工程隐喻）
- Notion（概念 → 已有词新用）

**生成方法**:
1. 选择 1-2 个核心词根（拉丁/希腊/英语）
2. 添加后缀（-io, -ly, -ify, -er）或前缀
3. 控制音节数（≤3 个音节）
4. 检查发音是否自然

### 框架 4: Compressed（压缩型）

**特征**: 缩写或首字母组合
**优点**: 简洁，现代感
**缺点**: 可能缺乏意义，记忆度低
**适用**: 技术产品，已有知名度的品牌扩展

**示例**:
- IBM（International Business Machines）
- AWS（Amazon Web Services）
- DJI（大疆创新）

**生成方法**:
1. 列出品牌核心价值词
2. 取首字母组合
3. 检查组合是否有不良含义
4. 评估是否需要全称来解释

**注意**: 新品牌不建议使用纯缩写，除非有强大的营销预算。

### 框架 5: Persona（人格型）

**特征**: 人名或角色感
**优点**: 亲和力强，易于建立情感连接
**缺点**: 可能限制品牌扩展
**适用**: 服务类产品，个人品牌

**示例**:
- Oscar（保险）
- Alexa（语音助手）
- 小红书（"小" + "红书" → 亲切 + 文化）

**生成方法**:
1. 确定品牌人格特征（友好？专业？幽默？）
2. 选择符合该人格的名字
3. 考虑跨文化接受度

### 框架 6: Poetic（诗意型）

**特征**: 文化、文学、自然引用
**优点**: 深度和内涵，差异化强
**缺点**: 可能需要解释，文化门槛
**适用**: 高端品牌，文化产品

**示例**:
- Quorum（法定人数 → 决策）
- Meridian（子午线 → 连接、精准）
- Atlas（擎天神 → 承载、地图）

**生成方法**:
1. 确定品牌希望关联的文化/自然概念
2. 研究神话、文学、历史、自然中的相关词汇
3. 评估目标用户是否能理解引用

### 框架 7: Geographic/Heritage（地理/ heritage 型）

**特征**: 地名或 heritage 指示
**优点**: 真实感，故事性
**缺点**: 可能限制全球化
**适用**: 有地域特色的产品，传统工艺

**示例**:
- Amazon（河流 → 广阔）
- Patagonia（地区 → 户外、探险）
- 茅台（地名 → 传统、品质）

**生成方法**:
1. 确定品牌与哪些地域/文化相关
2. 研究该地域的正面联想
3. 评估全球化时的接受度

## 1.3 Slogan 生成框架

### 五大 Slogan 原型

#### 1. Promise（承诺型）

**结构**: 品牌承诺交付的价值
**示例**: "Just do it." (Nike) / "Think different." (Apple)
**适用**: 成熟品牌，明确的价值主张
**生成方法**:
1. 确定品牌最核心的承诺
2. 用动词开头，简洁有力
3. 避免 generic（"We deliver quality"）

#### 2. Provocation（挑衅型）

**结构**: 挑战受众的现有信念
**示例**: "Think different." (Apple) / "Don't be evil." (Google 早期)
**适用**: 颠覆性品牌，挑战现状
**生成方法**:
1. 确定行业中的默认假设
2. 提出相反或挑战性的观点
3. 用命令式或反问式语气

#### 3. Pride（自豪型）

**结构**: 庆祝用户的身份
**示例**: "For the rest of us." (Apple 早期) / "Because you're worth it." (L'Oreal)
**适用**: 身份驱动的品牌
**生成方法**:
1. 确定目标用户的身份特征
2. 用第一或第二人称
3. 强调归属感和自豪感

#### 4. Contrast-Elevation（对比提升型）

**结构**: before/after 或 us/them 框架
**示例**: "When it absolutely, positively has to be there overnight." (FedEx) / "The ultimate driving machine." (BMW)
**适用**: 竞争激烈的品类
**生成方法**:
1. 确定与竞品的差异点
2. 用对比结构（不是...而是...）
3. 强调 superiority

#### 5. Concrete-Image（具体意象型）

**结构**: 具体的感官画面
**示例**: "15 minutes could save you 15% or more." (Geico) / "Melts in your mouth, not in your hands." (M&M's)
**适用**: 需要具体化抽象价值的品牌
**生成方法**:
1. 将抽象价值转化为具体场景
2. 使用数字、时间、感官词汇
3. 创造可想象的画面

### Slogan 节奏测试

**中文 Slogan 节奏**: 
- 最佳: 4+4 或 3+3 字结构（"Just do it" = "只管去做"）
- 次佳: 5+5 或 2+2+2 结构
- 避免: 超过 12 个字，或拗口的连续同音字

**英文 Slogan 节奏**:
- 最佳: ≤7 个词，有自然重音模式
- 测试: 大声朗读，是否感觉 forced？
- 避免: 被动语态，复杂从句

### Slogan-品牌契合测试

**测试问题**: "前 3 名竞争对手能用这个 Slogan 吗？"

```
示例:
Slogan: "Your code. Your flow. No ceremony."

测试:
- Jira 能用吗？不能 — Jira 就是 ceremony（流程、跟踪、报告）
- Linear 能用吗？勉强 — Linear 也强调简洁，但 "no ceremony" 不是其核心
- GitHub 能用吗？不能 — GitHub 是协作平台，不是个人 flow 工具

结论: 通过测试，有品牌独特性
```

## 1.4 双语命名一致性策略

### 三种双语命名模式

#### 模式 A: 中文主导 + 英文翻译

**特征**: 中文名是核心，英文名是翻译或音译
**示例**: 腾讯 (Tencent) / 阿里巴巴 (Alibaba)
**适用**: 中文市场为主，英文名为国际化准备

**策略**:
1. 先确定中文名，确保文化内涵和音韵
2. 英文名可以是：拼音（Tencent）、意译（ByteDance）、或谐音（Alibaba）
3. 确保英文名在目标国际市场无不良含义

#### 模式 B: 英文主导 + 中文音译/意译

**特征**: 英文名是核心，中文名是音译或意译
**示例**: Apple (苹果) / Microsoft (微软)
**适用**: 国际品牌进入中国市场

**策略**:
1. 先确定英文名，确保全球一致性
2. 中文名选择：音译（可口可乐）、意译（微软）、或混合（星巴克）
3. 中文名需通过 Six-Baseline 测试

#### 模式 C: 双名独立定位

**特征**: 中文名和英文名各自独立，针对不同市场
**示例**: 百度 (Baidu) / 京东 (JD.com)
**适用**: 中英文市场同等重要

**策略**:
1. 中文名和英文名分别按各自市场的定位策略生成
2. 确保两者在品牌核心价值上一致
3. 避免中文名和英文名在情感 register 上冲突

### 双语一致性检查清单

- [ ] 音韵一致性: 中文名和英文名读起来是否和谐？
- [ ] 语义一致性: 两者的核心含义是否一致？
- [ ] 情感一致性: 两者传达的情感 register 是否匹配？
- [ ] 文化一致性: 是否在两种文化中都没有负面联想？
- [ ] 视觉一致性: 如果同时出现，是否感觉像同一个品牌？
