# DOCX Quality Checklist

每次 .docx 交付前**逐条**走一遍。任何 ✗ 都不能交付。

## 1. 内容完整性

- [ ] 所有要求章节都已写入（与 brief 对照）
- [ ] 段落顺序符合逻辑结构
- [ ] 关键事实有出处或来源标注
- [ ] 无占位符（`TODO`、`[FILL IN]`、`Lorem ipsum`）残留
- [ ] 中文标点（。，：；）正确，不混用英文标点
- [ ] 内容改写没有引入未验证事实

## 2. 样式保真

详见 `style-inheritance.md`。

- [ ] 样式列表与原模板一致（无新增冗余样式）
- [ ] 标题层级（标题 1 / 2 / 3）正确递进
- [ ] 段落样式（"正文" / "引用"）使用一致
- [ ] 字体回退：嵌入字体或在交付说明中列出依赖
- [ ] 母版、字体、配色未被意外替换

## 3. 目录与编号

- [ ] TOC（目录）字段已 F9 刷新
- [ ] 页码连续，章节首页正确
- [ ] 列表编号（1. 2. 3.）正确，无中断
- [ ] 表格编号、图编号自动更新
- [ ] 脚注 / 尾注正常

## 4. 修订与批注

- [ ] 客户提交修订模式的文档：保留所有修订标记
- [ ] 批注按客户要求处理（resolve / delete 区分清楚）
- [ ] 无残留的"我的修订"未确认

## 5. 元数据

- [ ] 作者信息按要求设置（不泄露内部 ID）
- [ ] Track Changes 元数据已清理（`Inspect Document`）
- [ ] 文档属性（标题、关键字）正确填写

## 6. 格式细节

- [ ] 页眉页脚正确（奇偶页 / 首页设置）
- [ ] 页边距符合模板
- [ ] 字号 ≥ 12pt（印刷品要求）
- [ ] 行间距、段间距视觉一致
- [ ] 表格不异常断页

## 7. 图片与表格

- [ ] 图片清晰度 ≥ 300 DPI（如打印）
- [ ] 图片有 alt text（A11y）
- [ ] 表格有标题行（heading row repeat 已勾选）
- [ ] 跨页表格不会被错误截断
- [ ] 图片锚点合理

## 8. 链接与引用

- [ ] 所有 URL 链接可用（不是 `localhost` 或临时地址）
- [ ] 内部交叉引用（"见图 3"）正确指向
- [ ] 脚注、尾注编号正确

## 9. 兼容性

- [ ] 在 Word 2019 / 365 / LibreOffice 中打开正常
- [ ] PDF 导出正确（如需）
- [ ] Office for Mac 与 Windows 渲染差异（字体回退）

## 10. 交付包

- [ ] 文件名规范（含版本号或日期）
- [ ] 嵌入字体（如使用非标字体）
- [ ] 关联资源（图片、字体）一并交付
- [ ] 交付说明文档（CHANGELOG）说明改动
- [ ] 汇报中列出修改范围、验证结果、不可自动确认项

## 自动化辅助

```bash
# 列出所有样式
python -c "from docx import Document; d=Document('file.docx'); [print(s.name) for s in d.styles]"

# 检查作者元数据
python -c "from docx import Document; d=Document('file.docx'); print(d.core_properties.author)"
```

## 失败模式

- ❌ 用 LibreOffice 改后 Word 打开错位 → 用 python-docx 直接操作 XML
- ❌ TOC 不刷新 → 交付前手动 F9 或脚本触发
- ❌ 字体替换静默生效 → 嵌入字体或在 README 列出依赖
