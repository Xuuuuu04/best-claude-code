# PDF Quality Checklist

每次 PDF 交付（创建 / 编辑 / 提取摘要）前自检。

## 1. 内容准确性（最重要）

详见 `extraction-notes.md`。

- [ ] 摘要中"原文事实 / 推断 / 建议"三层已分离
- [ ] 每条事实带 `(p.{N})` 页码引用
- [ ] OCR 结果标注了置信度（如 `[OCR-LOW-CONFIDENCE]`）
- [ ] 表格已说明可能错位风险

## 2. 提取方式正确性

- [ ] 已识别 PDF 类型（文本型 / 扫描型 / 表单型）
- [ ] 用了对应的提取工具：
  - 文本型 → `pdftotext` / `pdfplumber`
  - 扫描型 → OCR (`tesseract` / `paddleocr` / Claude vision)
  - 表单型 → `pypdf` 的 form 字段 API
- [ ] 多栏 PDF 的列顺序已校验（不是从左到右纯文本）

## 3. 不可编辑限制说明

- [ ] 用户要求改扫描 PDF 文字 → 已说明需要重排版
- [ ] 用户要求修改签名 / 印章 → 已拒绝并升级到用户
- [ ] 不可逆操作（合并删除原文）→ 已确认

## 4. 引用规范

- [ ] 所有外部 PDF 内容引用都带 `(p.{N})`
- [ ] 摘要中明确区分"原文 vs 我的解读"
- [ ] 表格、图片都标注页码与位置

## 5. PDF 创建（如适用）

- [ ] 字号 ≥ 12pt（印刷品标准）
- [ ] 字体已嵌入（不依赖系统字体）
- [ ] 元数据正确（作者、标题、关键词）
- [ ] 书签 / 大纲生成（章节文档）
- [ ] 页码连续

## 6. PDF 编辑操作

- [ ] 旋转 / 合并 / 拆分 → 已用 `pypdf` / `qpdf`
- [ ] 水印 → 位置不遮挡关键内容
- [ ] 表单填写 → 字段值类型匹配
- [ ] 表单字段和权限限制未被误改

## 7. 交付包

- [ ] 文件名规范（含版本或日期）
- [ ] 文件大小合理（扫描件 < 5MB / 页 / 张）
- [ ] PDF/A 合规（如需归档）
- [ ] 元数据已清理敏感信息

## 8. A11y（如交付给用户）

- [ ] 文档结构有标记（headings、lists、tables）
- [ ] 图片有 alt text
- [ ] 表格有 header row 标记
- [ ] 阅读顺序正确（屏幕阅读器测试）

## 9. 汇报要求

- [ ] 列出页码范围、处理方式、验证结果
- [ ] 区分原文事实、推断和建议
- [ ] OCR 置信度低的部分单独提示用户

## 自动化辅助

```bash
# 识别 PDF 类型
pdftotext -layout input.pdf - | head -50  # 空白 = 扫描型

# 检查页数与基础元数据
pdfinfo input.pdf

# 提取表格
python -c "import pdfplumber; pdf=pdfplumber.open('input.pdf'); print(pdf.pages[0].extract_tables())"
```

## 失败模式

- ❌ 把扫描 PDF 内容当原文事实写 → 必须标注 OCR 置信度
- ❌ 多栏 PDF 串读为单栏 → 用 `pdfplumber` 配合坐标
- ❌ 修改扫描签名 → 立即拒绝（涉及伪造）
