# PDF Extraction Notes

PDF 三种类型的处理方式完全不同。错误判断会导致幻觉性输出（把 OCR 误识当原文事实）。

## 三类 PDF 识别表

| 类型 | 特征 | 提取方式 | 常见陷阱 |
|:--|:--|:--|:--|
| **文本型 PDF** | 可复制文字，PDF 内嵌文本层 | `pdftotext` / `pypdf2.extract_text()` / `pdfplumber` | 多栏、表格会被误读为顺序文本 |
| **扫描型 PDF** | 页面是图像，复制后只有空白或乱码 | OCR（`tesseract` / `paddleocr` / Claude vision） | OCR 会把"日"识成"曰"、"O"识成"0"，必须标注置信度 |
| **表单 PDF** | AcroForm / XFA 字段 | `pypdf` 的 form 字段 API；勿走文本 | 字段名 vs 字段值要分清，别把 label 当 value |

**先识别再提取**。命令：`pdftotext -layout input.pdf - | head -50`，若输出空白或乱码即扫描型。

## 关键纪律

### 1. 摘要必须区分三层信息

任何 PDF 摘要输出都要明确标注：

- **原文事实**（直接引用，给出页码）
- **推断**（基于原文得出的结论，标注"推断："）
- **建议**（给用户的行动建议，标注"建议："）

不允许把推断写成原文事实——这是 PDF 摘要任务最常见的幻觉模式。

### 2. 页码引用

每条事实必须带 `(p.{N})` 页码引用。多页同事实写 `(p.3-5)`。

### 3. 表格特殊处理

PDF 中表格被提取后行列错位是常态。处理顺序：

1. 用 `pdfplumber.extract_tables()` 优先而非 `extract_text()`
2. 提取失败回退到 vision（Claude 直接读图片）
3. 重要表格交付前**让用户对比原 PDF 一次**，承认不可百分百保真

### 4. OCR 标注置信度

OCR 结果在交付时必须标注：

```markdown
> [OCR-LOW-CONFIDENCE] "甲方应于 2026 年 5 月前支付" — 此段 OCR 置信度 0.78，"5" 可能是 "S"，建议人工核对原 PDF p.3
```

## 不可编辑限制

PDF 的"编辑"要分清能不能做：

| 任务 | 可行性 | 工具 |
|:--|:--|:--|
| 文本型 PDF 改文字 | 可（但格式可能错位） | `pikepdf` / Acrobat |
| 扫描型 PDF 改文字 | **不可** | 必须重排版 |
| 加水印 / 旋转 / 合并 / 拆分 | 可 | `pypdf` / `qpdf` |
| 填写表单字段 | 可（AcroForm） | `pypdf` |
| 修改扫描件中的签名 | **不可，且禁止**（涉及伪造） | 升级到用户 |
