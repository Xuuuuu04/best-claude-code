# 接单报价协议

## 核心原则

1. **数据驱动** — 所有定价基于2026年市场调研数据，不脑拍
2. **风险前置** — 报价前先评估风险，高风险项目加系数或拒绝
3. **透明定价** — 向甲方展示定价逻辑（技术栈 × 复杂度 × AI介入度）
4. **保护乙方** — 合同条款优先保护交付方权益

## 定价公式

```
报价 = 基准工时 × AI效率系数 × 复杂度系数 × 风险系数 × 时薪基准

其中：
- 基准工时：无AI时的传统估算（人天）
- AI效率系数：
  - 纯AI可生成（CRUD/简单页面）：0.3-0.5
  - AI辅助但需审查（业务逻辑）：0.6-0.8
  - AI难介入（架构/集成/调试）：1.0-1.3
- 复杂度系数：1.0（简单）- 2.5（极高）
- 风险系数：1.0（标准）- 1.5（数据敏感/合规/新技术/甲方不成熟）
```

## 报价模式选择

| 维度 | 固定价 | 工时报价（T&M） |
|:-----|:-------|:----------------|
| **适用场景** | 需求明确、范围可控、<3个月 | 需求模糊、AI项目、探索性强 |
| **风险承担** | 乙方（范围膨胀风险） | 甲方（成本不确定风险） |
| **2026年趋势** | 萎缩（AI使估算更难） | 增长（甲方愿为灵活性付费） |
| **溢价能力** | 低（被AI工具报价压低） | 高（专业判断力值钱） |
| **建议** | 加15-25%风险缓冲 | 加"not-to-exceed"上限 |

## 风险评估矩阵

| 风险因素 | 低(1.0) | 中(1.2) | 高(1.5) |
|:---------|:--------|:--------|:--------|
| **需求清晰度** | 有详细PRD | 有描述但缺细节 | 只有一句话需求 |
| **甲方成熟度** | 有技术对接人 | 非技术PM对接 | 完全非技术 + 高期望 |
| **技术栈熟悉度** | 熟练 | 用过但需查文档 | 全新技术栈 |
| **数据敏感度** | 公开数据 | 内部业务数据 | 金融/医疗/政府 |
| **合规要求** | 无特殊要求 | GDPR/等保 | EU AI Act + 行业监管 |
| **付款历史** | 有平台托管/预付款 | 月结 | 后付/无合同 |

**一票否决项**（任何一条 → 建议不接或大幅提高报价）：
- 无合同或口头协议
- 要求"先做完看效果再付款"
- 需求极度模糊且拒绝补充
- 涉及违法或灰色地带

## 付款节点建议

| 项目类型 | 定金比例 | Milestone结构 | 尾款期限 |
|:---------|:---------|:--------------|:---------|
| 小型（<$5,000 / <¥3万） | 50%预付 | 50%-50% | 验收后7天 |
| 中型（$5k-50k / ¥3-30万） | 30-40%预付 | 30%-30%-30%-10% | 验收后14天 |
| 大型（>$50k / >¥30万） | 20-30%预付 | 月度milestone | 验收后30天 |
| AI/Agent项目 | 40-50%预付 | 增加"PoC验证"节点 | 验收后14天 |

## 合同关键条款检查清单

| # | 条款 | 必须包含 | 建议措辞 |
|:--|:-----|:---------|:---------|
| 1 | **AI使用披露** | 是 | "乙方须披露所有使用的AI工具清单，交付物经人工审查" |
| 2 | **知识产权** | 是 | "成果归甲方，AI生成内容须经审查并保留审查记录" |
| 3 | **验收标准** | 是 | 功能清单 + 测试用例 + 代码审查标准，书面确认 |
| 4 | **Scope变更** | 是 | "超出SOW范围的需求须书面变更单，另行报价" |
| 5 | **付款节点** | 是 | 预付比例 + milestone定义 + 逾期罚息（1.5-2%/月） |
| 6 | **违约金上限** | 是 | 不超过合同总额的20-30% |
| 7 | **保密协议** | 视情况 | 数据接触程度决定是否单独NDA |
| 8 | **争议解决** | 是 | 明确仲裁地（国内/国际）和适用法律 |

|:-----|:---------|:-----|:-----|:-----|
| ... | ... | ... | ... | ... |
| **总计** | | | | **{总价}** |

## 5. 付款节点
{按付款节点建议填写}
