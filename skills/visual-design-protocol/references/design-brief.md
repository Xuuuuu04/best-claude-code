# Design Brief Template

设计开始前**必须**填的 brief。开始造之前先确认这些。

## 1. 用户与场景

- **目标用户**：具体角色，不是"用户"。例：财务总监查看月报 / 工程师 review PR
- **使用场景**：什么环境、什么设备、什么时间。例：办公桌 27 寸屏 / 通勤地铁 6 寸手机
- **关键任务**：用户来这里做的 1-2 件事
- **频率**：日常 / 每周 / 每月 / 一次性

## 2. 情绪与气质

| 维度 | 选项 |
|:--|:--|
| 形式感 | 严肃 / 中性 / 活泼 |
| 信息密度 | 密集 / 平衡 / 稀疏 |
| 视觉权重 | 厚重 / 轻盈 |
| 节奏 | 紧凑 / 呼吸感 / 戏剧化 |
| 表达 | 内敛 / 自信 / 大胆 |

## 3. 差异化点

- 与同类产品（列 2-3 个）的视觉差异是什么？
- 哪些视觉决策是为了和竞品**反着来**？
- "如果删掉所有装饰，这个设计还有辨识度吗？"

## 4. 禁区（硬约束）

- 不能用的颜色（品牌冲突色）
- 不能用的字体
- 不能用的视觉元素
- 不能违反的 A11y 红线

## 5. 必有元素

- 品牌 logo / wordmark
- 法律 / 合规元素（footer 备案号、条款）
- 必须支持的语言 / RTL
- 必须适配的最小屏幕

## 6. 实现约束

- 技术栈（React / Vue / Svelte / 原生）
- 已有组件库（Radix / shadcn / Ant Design）
- 性能预算（首屏 LCP < X ms、bundle < Y kb）
- 浏览器支持范围

## 7. 状态矩阵（必填）

每个交互组件都要设计：
default / hover / focus / active / disabled / loading / empty / error / success

## 8. 验证方式

- 截图对比设计稿（pixel-level）
- A11y 工具自动测试（axe / Lighthouse）
- 真机测试（具体型号清单）
- 用户访谈（数量 + 关键问题）

## 9. 不确定项（先列出）

设计开始前模糊的部分。用 AskUserQuestion 与用户确认，**不要假设**：

- [ ] 是否需要暗色模式
- [ ] 是否需要变体（数量 + 维度）
- [ ] 哪些是 MVP 必有 vs 后续 nice-to-have
- [ ] 设计风格倾向（导向用户提供参考）

## 完整性自检

填完 brief 后回答：

- 能给设计师一句话描述这个产品吗？
- brief 里有"等等再说"的项吗？（有 → 不要开始）
- 验收标准是可测试的吗？
- 风险点已识别吗？

只有全部 ✅ 才进入设计阶段。
