# Formula Rules

Excel 公式比单元格值重要。把公式静默替换成静态值是 xlsx 交付最常见的破坏性错误。

## 核心铁律

### 1. 公式不可静默替换为静态值

```
❌ 错误：cell.value = 100   # 原本是 =SUM(A1:A10)
✅ 正确：保留公式，仅修改公式参数或依赖区域
```

例外：用户**显式**要求"求出结果后清空公式"才允许。即使如此也要在交付说明里明确写："已按要求将 N 个公式替换为静态值"。

### 2. 相对引用 vs 绝对引用

| 写法 | 含义 | 何时用 |
|:--|:--|:--|
| `A1` | 完全相对 | 拖动时行列都偏移，多用于公式池 |
| `$A$1` | 完全绝对 | 引用固定锚点（汇率、税率） |
| `A$1` | 行绝对 | 多列共享首行参数 |
| `$A1` | 列绝对 | 多行共享首列参数 |

**复制公式前先确认**：原作者用了哪种引用？盲目改会让整张表脏掉。

### 3. 财务模型三区分离

财务模型的可读性 = 输入区/计算区/输出区清晰分离：

```
┌─ Inputs（手填）─┐
│  增长率: 0.05    │
│  贴现率: 0.08    │
└─────────────────┘
        ↓
┌─ Calculations（公式）─┐
│  =Inputs!B1*A2        │
└──────────────────────┘
        ↓
┌─ Outputs（汇总）─┐
│  =SUM(Calc!B:B)  │
└─────────────────┘
```

修改财务模型时不要在计算区直接覆盖数字——把它改回输入区。

### 4. 跨表引用与命名区域

- 优先用**命名区域**（`Tax_Rate`）而非 `Sheet2!$B$3`，可读性高且不易因增删行失效
- 跨表引用必须确认目标表存在；新建表前先 `wb.sheetnames`
- `INDIRECT()` 慎用——它阻断 Excel 的依赖追踪

### 5. 数组公式与 LET / LAMBDA

- `=SUMPRODUCT(...)` 优于老式 `{=SUM(IF(...))}`（CSE 数组公式）
- Excel 365 用户可用 `LET` / `LAMBDA` / `XLOOKUP`，但**先确认目标版本**
- 客户用 Excel 2019 时不要写 `XLOOKUP`，回退 `INDEX/MATCH`

## 验证清单

- [ ] 所有原有公式仍是公式（未被静默替换）
- [ ] 引用类型（绝对/相对）符合模板原意
- [ ] 跨表引用未失效（重命名 Sheet 后特别注意）
- [ ] 公式中无硬编码"魔法数"（应回到输入区）
- [ ] 已用 `wb.calculate()` 触发重算或 Excel 中按 F9
- [ ] CSV 导出时公式被替换为值（这是 CSV 限制，必须告知用户）

## 重算验证

修改后**一定要重算并对比关键输出**：

```python
import openpyxl
wb = openpyxl.load_workbook(path, data_only=False)
# 修改公式
wb.save(path)

# 用 LibreOffice 命令行触发重算
# soffice --headless --calc --convert-to xlsx path
```

`openpyxl` 不会自动算公式结果——它只读写公式字符串。要看结果必须用 `data_only=True` 或外部工具重算。

## 失败模式

- ❌ 改完表格 SUM 全是 0：因为新插入的行没被纳入 SUM 范围（用 SUM(A:A) 整列引用避免）
- ❌ 复制公式后引用全错：相对/绝对没改对
- ❌ 客户打开报"循环引用"：通常是命名区域指向了引用自身的单元格
- ❌ 数据透视表数据源失效：源表行数变了未刷新
