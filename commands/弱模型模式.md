---
description: 切换项目到弱模型模式（适配 GLM / MiniMax / DeepSeek / Doubao / step 等弱模型），收紧主进程判断空间、强制命令化路径、前移 compact 阈值。用法：/弱模型模式 on | off | status。
argument-hint: [on / off / status，默认 status]
---

<command>

<section id="rules">
<title>铁律</title>
<content>
1. **Project CLAUDE.md must exist**: Cannot activate weak-model mode without an existing project CLAUDE.md. If absent, instruct user to run /项目分析 first.
2. **Only project CLAUDE.md is modified**: This command writes ONLY to the current project's CLAUDE.md frontmatter. `~/.claude/` core files are never touched.
3. **Mode switch does not retroact**: Tasks already in progress are not retroactively affected. The new constraints apply from the next dispatch onwards.
4. **Current directory must be a project**: No .git or equivalent marker → refuse and ask user to cd into project root.
5. **Status is the default action**: If `$ARGUMENTS` is not `on` or `off`, show current mode status. Never silently do nothing.
</content>
</section>

<section id="scope">
<title>本 command 覆盖范围</title>
<content>
**适用模型**：GLM / MiniMax / DeepSeek / Doubao / step 等弱推理能力模型

**启用后强制项**（弱模型模式 on）：
1. DispatchPlan 6 项全部必填，缺项拒绝派发
2. 快速路径（output-style §7）禁用；所有任务走完整流水线
3. 单次 Agent 输入 ≤ 2K token（超限自动拆子任务）
4. Compact 触发阈值：60%（默认 80%，此档前移）
5. Agent charter 注入 core 段（若该 Agent 支持 core-slim 标注）
6. ★ Insight 四要素必须齐全；Hook-D 物理拦截
7. 优先走 command 入口（/需求蒸馏 /新功能 /快速修复 /安全检查 /代码审查 /会话交接）

**关闭后**：回到 production 默认档，以上约束解除
</content>
</section>

<section id="dispatch-flow">
<title>调度流水线</title>
<content>
**Step 1 — 解析参数**（主进程执行）
- 读 `$ARGUMENTS`：on / off / status（默认 status）
- 验证：当前目录有 .git 等项目标记；否则 STOP

**Step 2a — 若 action = "on"**
- 检查项目 CLAUDE.md 存在；否则 STOP，提示先运行 /项目分析
- 写入 CLAUDE.md frontmatter：
  ```yaml
  project-mode: weak-model
  weak-model-enabled-at: {YYYY-MM-DD HH:mm}
  ```
- 向用户输出启用通告（强制项 7 条 + 推荐做法）

**Step 2b — 若 action = "off"**
- 从 CLAUDE.md frontmatter 移除 `project-mode: weak-model`（或改为 production）
- 输出：弱模型模式已关闭，回到 production 默认档

**Step 2c — 若 action = "status" 或参数为空**
- 读 CLAUDE.md frontmatter
- 输出当前 project-mode 值 + 启用时间（若有）
</content>
</section>

<section id="output-contract">

<content>
启用后输出：

```
2. 快速路径禁用
3. 单次 Agent 输入 ≤ 2K token
4. Compact 阈值：60%

- 多用 /会话交接 固化状态
```

状态查询输出：
```
当前项目模式：{project-mode 值}
启用时间：{weak-model-enabled-at，若有}
```
</content>
</section>

<section id="final_reminder">
<title>Final Reminder</title>
<content>
- Only the project CLAUDE.md frontmatter is modified. ~/.claude/ files are never touched.
- No project marker = refuse. Must be inside a project directory.
- Mode switch applies from next dispatch — does not affect tasks already in progress.
- Default action (no args) is status check, not activation.
- Weak-model mode significantly tightens Harness constraints — communicate this clearly to the user.
</content>
