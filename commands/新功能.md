<section id="rules">
<title>铁律</title>
<content>
1. **Strict serial execution (Hook-E enforced)**: Each step dispatches exactly one Agent. The next step does not start until the current one completes. Hook-E physically blocks parallel dispatch.
2. **Quality gates are non-negotiable**: Steps 7 (test-func), 9 (test-lead) and 12 (client handoff) are MANDATORY and cannot be skipped for any reason, including schedule pressure.
3. **Conditional steps skip must be documented**: Steps 4 (creative/tech-research), 5 (architect), 7 (security-auditor), 10 (visual-designer) may be skipped only with an explicit reason logged in progress-log.md and ★ Insight.
4. **Mid-stream direction change = stop**: If requirements change significantly mid-pipeline, STOP the entire pipeline and restart with /需求蒸馏. Do not force-push through with an outdated spec.
5. **Failure at any step stops the pipeline**: Do NOT auto-proceed past a failure. Report to user and wait for explicit instruction.
</content>
</section>

<section id="scope">
<title>本 command 覆盖范围</title>
<content>
**适用场景**：
- 新业务能力（用户功能 / 新 API / 新页面 / 新集成）
- 需要拆多个 Task（跨文件、跨模块）
- 首次上线前进入生产环境

**不适用**（用其他命令）：
- 明显 bug → /快速修复
- 纯重命名/格式化 → 直接派开发 Agent
- 单文件 + < 20 行 + 不涉接口 → 考虑 /快速修复

**预期时长**：
- 小特性：6-10 跳，30-60 分钟
- 中特性：12-20 跳，1.5-3 小时
- 大特性：20+ 跳，建议拆多个 /新功能
</content>
</section>

<section id="dispatch-flow">
<title>调度流水线（12 步）</title>
<content>
**阶段 A — 需求侧**

**Step 1 — 需求语义增强**（派 客户沟通师 Agent，必须）
- 识别真实业务目标 / 显式要求 / 隐含假设 / 边界问题

**Step 2 — Task 拆分**（派 项目管理师 Agent，必须）
- INVEST 原则拆分 Task / 优先级 / 依赖关系 / 状态管理

**Step 3 — 范围收紧**（派 开发组长 Agent，必须）
- In-scope / Out-of-scope 锁定 / 文件级拆分 / 接口契约初稿

**Step 4 — 可选支撑**（条件触发，任一需要时）
- 创意策划师：命名 / 文案
- 技术调研师：技术选型
- 深度研究员：深度研究

**阶段 B — 设计侧**

**Step 5 — 架构设计**（条件触发，满足任一时：跨模块 / 影响核心数据流 / 引入新技术栈 / 超过 3 个子模块）
- 派 架构师 Agent
- 产出：ADR + 数据模型 + 接口契约
- 不触发时在 ★ Insight 注明跳过理由

**阶段 C — 实现侧**

**Step 6 — 开发实现**（每次只派一个，完成一个再派下一个，Hook-E 强制）
- 候选：backend / frontend / database / miniprogram-dev / ml-engineer

**阶段 D — 审查侧（全部不可跳）**

**Step 7 — 代码审计师**（必须）
- per-diff 代码质量 + 安全基线 4 项

**Step 8 — 安全审计师**（条件触发：涉认证/授权/加密/支付 / 引入新依赖 / 临近上线）
- OWASP Top 10 + 依赖 CVE

**Step 9 — 功能测试师**（必须）
- 端到端用户流程 + 边界值 + 异常路径

**Step 10 — 界面测试师**（涉前端改动时，触发后不可跳）
- 截图 + 交互校验

**阶段 E — 发布侧**

**Step 11 — 视觉设计师**（可选：UI 需要设计系统级调整时）

**Step 12 — 运维部署工程师**（必须）
- 部署配置 + CI/CD + 可观测性最小集

**Step 13 — 客户沟通师**（必须）
- 交付沟通 + 售后跟进预案

**最终闸门 — 测试总监师裁决**（所有步骤通过后）
- 通过 / 有条件通过 / 打回
</content>
</section>
