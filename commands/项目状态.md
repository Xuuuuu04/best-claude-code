<command>

<section id="rules">
<title>铁律</title>
<content>
1. **Read-only, zero file writes**: This command MUST NOT write, edit, or delete any file. It is purely observational.
2. **Must be in a project**: If not inside a project directory (no .git or equivalent), refuse and prompt user to cd into project root or run /项目分析.
3. **Output stays under 1 screen**: Do not dump entire TASK.md or progress-log.md. Summarize. If Tasks > 10, show Top 5 in-progress + Top 5 waiting.
4. **Proactively surface risks**: Tasks with 2+ rework rounds or 7+ days without movement must be highlighted — do not wait for user to ask.
5. **No dispatch or recommendations inside this command**: Just report current state. Do not issue dispatch suggestions or next-step recommendations (that is pm's job).
</content>
</section>

<section id="scope">
<title>本 command 覆盖范围</title>
<content>
**读取来源**：
- `tasks/TASK.md`：所有 Task 状态
- `progress-log.md` 最近 30 行
- 项目根 CLAUDE.md 的 `## 最近进度` 头 5 条

**展示内容**：
- 当前项目模式（poc / production / debug / weak-model）
- 活跃 Task 数量
- 进行中 Task（Task ID / 标题 / 状态 / 负责 Agent / 阻塞）
- 待开始 Task（Task ID / 标题 / 优先级）
- 最近完成（近 5 条）
- 最近调度记录（progress-log 近 5 条）
- 风险/阻塞提示
</content>
</section>

<section id="dispatch-flow">
<title>调度流水线</title>
<content>
**Step 1 — 项目识别**（主进程执行）
- 确定当前工作目录对应项目
- 若不在任何项目内 → 提示用户 cd 到项目或先跑 /项目分析

**Step 2 — 数据采集**（主进程执行，只读）
- 读 tasks/TASK.md：所有 Task 状态
- 读 progress-log.md 最近 30 行
- 读项目根 CLAUDE.md 的 `## 最近进度` 头 5 条

**Step 3 — 输出报告**（主进程执行）
- 按模板格式化输出（见 output-contract）
- 若无 TASK.md → 提示"项目尚未初始化 Task 文件，建议运行 /需求蒸馏"
- 主动标记风险：超 2 轮返工 / 超 7 天无动作的 Task
</content>
</section>

<section id="output-contract">
<title>产出合约</title>
<content>
```
## 项目状态：{项目名}

- 当前模式：{poc / production / debug / learning / weak-model}
- 活跃 Task 数：{N}
- 最近进度：{日期 + 标题}

### 进行中
- [Task-XXX] {标题}   状态：{state}   负责：{Agent}   阻塞：{如有}

### 待开始
- [Task-XXX] {标题}   优先级：{高/中/低}

### 最近完成（近 5 条）
- [Task-XXX] {标题}   完成：{日期}

### 最近调度（近 5 条）
- {时间} {谁 → 谁} {操作} {TaskID} {理由}

### 风险提示
- {超 2 轮返工 / 超 7 天无动作的 Task 主动点名}
```
</content>
</section>
