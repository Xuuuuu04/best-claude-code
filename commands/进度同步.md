---
description: 将当前会话上下文、进度、决策、变更同步写入项目各层 CLAUDE.md 的"最近进度"段，让下次会话任意层级打开都能回到上下文。触发信号：阶段性工作完成、要下班前、会话 context 接近饱和、完成关键决策。
argument-hint: [可选：本次进度的关键摘要，将附加到进度记录]
---

<command>

<section id="rules">
<title>铁律</title>
<content>
1. **Only write to project CLAUDE.md files**: Hook-A blocks writes to `~/.claude/` core files. This command only updates CLAUDE.md files within the current project directory.
2. **Append only, never overwrite**: Only append to the `## 最近进度` section. Never delete or overwrite existing content in CLAUDE.md.
3. **Existing CLAUDE.md only**: Do not create new CLAUDE.md files. If a directory has no CLAUDE.md, skip it — use /项目分析 first.
4. **No debug/temp info**: Progress log entries must contain meaningful information (decisions, changes, blockers, todos) — no temporary debugging notes.
5. **Single entry ≤ 15 lines**: Each progress entry must stay concise. If more detail is needed, link to a separate document.
</content>
</section>

<section id="scope">
<title>本 command 覆盖范围</title>
<content>
**写入目标**：项目内（含 .git 标记）已存在的 CLAUDE.md

**提取维度**：
- 已完成的变更（文件增/删/改）
- 架构决策（做了什么决定、为什么）
- 当前任务进度（正在做什么、完成到哪）
- 待办/阻塞（剩余步骤、被什么阻塞）
- 已知问题（bug / 技术债 / 风险）
- 关键上下文（后续开发必须知道的）

**不写入**：~/.claude/ 核心目录（Hook-A 物理拦截）
</content>
</section>

<section id="dispatch-flow">
<title>调度流水线</title>
<content>
**Step 1 — 收集上下文**（主进程执行）
- 从当前会话提取 6 个维度（见 scope）
- 将 `$ARGUMENTS` 作为关键摘要补充

**Step 2 — 定位所有 CLAUDE.md**（主进程执行）
- Glob 扫描项目内所有 CLAUDE.md（排除 ~/.claude/ 下的）

**Step 3 — 分层适配写入**（主进程执行）
- 根目录层：当前里程碑 / 全局架构变更 / 跨模块影响 / 全局待办与风险
- 模块级：本模块文件变更 / 接口变更 / 模块待办 / 相关架构决策
- 子模块级：具体文件变更 / 函数/类级修改 / 本子模块 bug 修复

每层 CLAUDE.md 末尾追加 `## 最近进度`，时间倒序，格式：
```markdown
### {YYYY-MM-DD HH:mm} — {简短标题}
- **变更类型**：{新增 / 修复 / 重构 / 配置 / 文档}
- **影响范围**：{本模块 / 跨模块 - 哪些}
- **变更内容**：{项}
- **架构决策**：{如有 rationale}
- **待办**：{如未完成}
- **关联**：{Task ID / Issue / PR}
```

**Step 4 — 同步 Memory**（主进程执行，条件：存在长期价值信息）
- 写入 `~/.claude/projects/{project}/memory/`（Hook-A 白名单允许）
- 更新 MEMORY.md 索引

**Step 5 — 汇报**
- 输出：更新了 N 个 CLAUDE.md / 根目录摘要 / Memory 同步结果

**条目保留规则**
- 模块/子模块级：≤ 10 条，超出归档到同级 progress-archive.md
- 根目录级：≤ 20 条
</content>
</section>

<section id="final_reminder">
<title>Final Reminder</title>
<content>
- Append only — never delete or overwrite existing CLAUDE.md content.
- Only update CLAUDE.md files that already exist — do not create new ones.
- ~/.claude/ core files are off-limits (Hook-A blocks them).
- Each progress entry stays under 15 lines — link to detail files for more.
- Layer the content appropriately: root gets cross-module impact, module gets local changes.
