---
description: 深度分析整个项目目录树，为每一层生成/更新 CLAUDE.md（目录层级、职责、技术栈、架构、接口、逻辑概述、文件索引）。触发信号：新接手项目、首次阅读陌生 codebase、架构发生重大变化后需要全量重建知识图谱。
argument-hint: [可选：目标目录路径，默认当前工作目录]
---

<command>

<section id="rules">
<title>铁律</title>
<content>
1. **Never write to `~/.claude/` core files**: Hook-A physically blocks writes to `~/.claude/` (except whitelisted paths). This command operates on project directories only.
2. **Empty argument + home directory = STOP**: If `$ARGUMENTS` is empty AND the current directory is `~` or `~/.claude`, refuse and ask user to specify a project directory or `cd` into one first.
3. **Project marker required**: Target directory must contain at least one of `.git` / `package.json` / `pyproject.toml` / `Cargo.toml` / `go.mod`. Warn and ask before proceeding without one.
4. **Read before write**: Existing CLAUDE.md files MUST be read before updating. Content inside `<!-- user-content -->` markers must be preserved unchanged.
5. **Single CLAUDE.md ≤ 200 lines**: If content would exceed 200 lines, trim the file index section. Quality over quantity.
</content>
</section>

<section id="scope">
<title>本 command 覆盖范围</title>
<content>
**分析目标**：项目目录树下所有含代码的目录

**排除目录**：node_modules / .git / dist / build / __pycache__ / .next / target / vendor / .venv / venv / coverage / .turbo / .claude / .config / .vscode / .idea

**每层 CLAUDE.md 产出**：目录职责 / 技术栈 / 架构 / 对外接口 / 内部逻辑 / 依赖关系 / 文件索引

**不触碰**：业务代码（只创建/更新 CLAUDE.md）
</content>
</section>

<section id="dispatch-flow">
<title>调度流水线</title>
<content>
**Step 1 — 边界校验**（主进程执行，不派 Agent）
- 确定 target = `$ARGUMENTS` 或当前目录
- 校验 B1（非 ~/.claude/）/ B2（非 ~ 空参数）/ B3（有项目标记）
- 未通过 → STOP，向用户说明原因

**Step 2 — 目录树扫描**（主进程执行）
- 用 Glob 扫描 target 下所有含代码目录（排除清单见上）
- 产出：目录清单 + 每个目录是否已有 CLAUDE.md 的标记

**Step 3 — 深度探索**（并行 Explore Agent，最多 5 个一批，GP-O12 豁免授权：纯只读，无写依赖）
- 每个 Agent 任务：列出文件 / 识别语言和框架 / 分析入口文件 / 提取对外接口 / 理解内部逻辑 / 识别调用关系

**Step 4 — 逐层写入 CLAUDE.md**（主进程执行）
- 模板内容（每层）：目录层级 / 目录职责 / 技术栈 / 架构 / 对外接口 / 内部逻辑概述 / 依赖关系 / 扩展方向 / 文件索引
- 已有 CLAUDE.md 先读再改，保留 `<!-- user-content -->` 标记内容
- 已完善的只做增量补充

**Step 5 — 项目根统领**（主进程执行，仅项目根目录）
- 额外产出：项目总览 + ASCII 目录树 + 全局技术栈汇总 + 模块依赖图

**Step 6 — 汇报**
- 输出：扫描 N 个目录 / 新增 X / 更新 Y
</content>
</section>

<section id="final_reminder">
<title>Final Reminder</title>
<content>
- Never modify business code — only create/update CLAUDE.md files.
- Read existing CLAUDE.md before writing — never blindly overwrite user content.
- Empty args + home directory = refuse and ask for explicit path.
- Single CLAUDE.md file stays under 200 lines.
- Parallel Explore Agents (Step 3) are read-only — this is the only exception to the serial rule, under GP-O12 waiver.
