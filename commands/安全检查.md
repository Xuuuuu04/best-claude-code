<section id="rules">
<title>铁律</title>
<content>
1. **High-severity findings block release**: Any unresolved HIGH severity finding MUST block release. test-lead cannot issue a pass verdict while high-severity findings remain open.
2. **Audit report must be committed to git**: Security audit reports are compliance artifacts. They MUST be saved to `security-audits/audit-{YYYY-MM-DD}.md` and committed.
3. **No "PoC exemption"**: A proof-of-concept or demo codebase is NOT exempt from security baseline. Output-style §1.3 explicitly requires GP-S* compliance for all code modes.
4. **Re-audit after high-severity fix**: High-severity findings, once fixed, MUST be re-verified by security-auditor (not code-review). The same Agent that flagged it must clear it.
5. **Cannot skip this gate at milestones**: /安全检查 is mandatory before any production deployment or milestone cut. pm must document skip reason in progress-log if it is explicitly waived.
</content>
</section>

<section id="scope">
<title>本 command 覆盖范围</title>
<content>
**触发场景**
- 里程碑前（即将 cut release branch）
- 上线前（生产环境部署前）
- 引入新依赖批次（一次 3+ 个新依赖）
- 触碰核心数据流（认证/授权/加密/支付/PII）
- 合规要求（SOC2 / GDPR / 等保 / 金融监管）
- 安全事件响应（被报告漏洞 / 怀疑已泄露）

**审计范围确定**：`$ARGUMENTS` 指定模块 → 按指定范围；否则 → 全量 + 最近 30 天提交
</content>
</section>

<section id="dispatch-flow">
<title>调度流水线</title>
<content>
**Step 1 — 确定审计范围**（主进程执行）
- 解析 `$ARGUMENTS`
- 收集：项目技术栈 / 部署模式 / 是否 public-facing / 数据敏感度分级 / 相关 Task/ADR

**Step 2 — 深度安全审计**（派 安全审计师 Agent，必须）
- 方法论：OWASP Top 10 + CWE Top 25 + 依赖 CVE + 密钥扫描 + 合规核对
- 产出报告结构：
  - 审计摘要（范围/日期/方法论）
  - 高危 findings（必须修复）：[F-001] 漏洞名 | 位置：文件:行 | CWE 编号 | 描述 | 利用场景 | 修复建议
  - 中危 findings（上线前修复）
  - 低危/加固建议
  - 合规差距
  - 依赖 CVE 报告
  - 密钥泄露检查（git log + HEAD）

**Step 3 — 高危修复**（条件：存在高危 findings，派 /快速修复）
- 每个高危 finding 单独走一次 /快速修复
- 修复完成后回到 security-auditor 复核（不是 code-review）

**Step 4 — 裁决**（派 测试总监师 Agent，必须）
- 全高危清零 + 全中危修复或明确豁免 → 通过
- 有高危 → 打回，明确禁止上线
- 中危不影响本次上线范围 → 有条件通过（含豁免期）

**Step 5 — 归档报告**（主进程执行）
- 保存到 `security-audits/audit-{YYYY-MM-DD}.md`
- git commit（合规追溯）
- 输出最终结论摘要
</content>
</section>

- High-severity findings block release — no exceptions.
