description: 会话结束前状态 dump：把进度、未决问题、下次入口写入 progress-log 和 NEXT-SESSION.md，并同步 Memory。触发信号：context > 70%、要下班、完成重要里程碑、切换项目前。
argument-hint: [可选：本次会话的关键备注，将原样附加到交接记录]

<command>

<section id="rules">
<title>铁律</title>
<content>
1. **Project directory required**: Must execute within a project directory (has .git or equivalent marker). If in `~` or `~/.claude`, refuse and ask user to specify project root.
2. **Never write to ~/.claude/ core directories**: Hook-A blocks writes to agents / output-styles / shared / etc. Only project-level files and `~/.claude/projects/` memory paths are writable.
3. **Append only**: progress-log.md is append-only. Never delete or modify existing entries.
4. **NEXT-SESSION.md ≤ 60 lines**: Must stay compact enough for a weak model to parse at session start. Over 60 lines = trim to essentials.
5. **Honest about uncertainty**: If the current state is unclear (e.g., after compact), write what is known and mark unknowns explicitly. Never fabricate status.
</content>
</section>

<section id="scope">
<title>本 command 覆盖范围</title>
<content>
**触发时机**：context > 70% / 要下班 / 完成重要里程碑 / 切换项目前

**不适用**：只做了小修改且不需要持久化状态

**提取信息**：
- 本次会话完成了什么（时间序）
- 还没完成的待办（按优先级）
- 当前阻塞（谁等谁/等什么决定）
- 做过的关键决策（含 rationale）
- 未解决的问题清单
- 下一步推荐动作（派谁/做什么/预期产出）
- 弱模型容易踩的坑（特别提醒）
</content>
</section>

<section id="dispatch-flow">
<title>调度流水线</title>
<content>
**Step 1 — 确定目标项目**（主进程执行）
- 取当前工作目录
- 若在 ~ 或 ~/.claude 等非项目目录 → 拒绝，要求用户指定

**Step 2 — 收集会话关键信息**（主进程执行）
- 来源：当前会话 + progress-log.md + tasks/（即使 context 已 compact）
- 提取上方 7 个维度

**Step 3 — 写 progress-log.md**（主进程执行，追加模式）
```markdown
---
## {YYYY-MM-DD HH:mm} · 会话交接

### 本次完成
- {列表}

### 未决待办
- [ ] {列表}

### 阻塞
- {描述}

### 关键决策
- {列表}

### 下次入口
- 建议先派：{Agent} 做 {什么}
- 阅读：{关键文件路径}
- 避免：{踩过的坑}

### 用户备注
{$ARGUMENTS 原样附上，若有}
```

**Step 4 — 写 NEXT-SESSION.md**（主进程执行，覆盖项目根，≤ 60 行）
- 当前状态一句话
- 立即要做的下一件事（派谁/做什么/预期产出）
- 必须先读的文件（≤ 3 个）
- 已知坑（弱模型容易犯的错）
- 日期/交接人

**Step 5 — 同步 Memory**（主进程执行，条件：有长期价值信息）
- 写入 `~/.claude/projects/{project}/memory/session-handoff-{YYYY-MM-DD}.md`
- 更新 MEMORY.md 索引（Hook-A 白名单允许写 ~/.claude/projects/）

**Step 6 — 汇报**
- progress-log.md 追加 1 段 / NEXT-SESSION.md 更新 / Memory 同步结果 / 下次入口一句话
</content>
</section>

<title>Final Reminder</title>
<content>

- NEXT-SESSION.md stays under 60 lines — trim ruthlessly.
- ~/.claude/ core files are off-limits. Only project files and ~/.claude/projects/ memory are writable.
